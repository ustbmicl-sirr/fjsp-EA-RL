# 📊 测试结果目录

这个目录用于存放系统测试产生的各种结果文件。

## 📁 目录结构

```
test_results/
├── html/                    # HTML可视化文件
│   └── disjunctive_graph_final.html
├── reports/                 # 测试报告
└── README.md               # 本说明文件
```

## 🎨 HTML可视化文件

### html/目录
存放系统生成的交互式HTML可视化文件：

- **disjunctive_graph_final.html**: 析取图可视化结果
  - 交互式节点和边
  - 支持缩放和平移
  - 悬停显示详细信息
  - 基于Plotly.js技术

## 📋 测试报告

### reports/目录
存放各种测试报告文件：

- 性能基准测试报告
- 算法对比分析报告
- 系统功能测试报告
- 用户体验测试报告

## 🔧 文件管理

### 自动生成
大部分文件由系统自动生成：
- 运行性能测试时自动创建报告
- Web界面操作时生成可视化文件
- 算法求解时产生结果文件

### 手动清理
定期清理旧的测试文件：
```bash
# 清理HTML文件（保留最新的）
find html/ -name "*.html" -mtime +7 -delete

# 清理报告文件（保留最新的）
find reports/ -name "*.md" -mtime +30 -delete
```

## 📊 文件类型说明

### HTML可视化文件
- **格式**: .html
- **技术**: Plotly.js + HTML5
- **特点**: 交互式、可缩放、响应式
- **用途**: 结果展示、报告嵌入、演示

### 测试报告
- **格式**: .md, .json, .csv
- **内容**: 性能数据、测试结果、分析图表
- **用途**: 性能分析、问题诊断、改进建议

### 数据文件
- **格式**: .json, .csv, .xlsx
- **内容**: 原始测试数据、统计结果
- **用途**: 数据分析、趋势追踪、对比研究

## 🎯 使用建议

### 查看结果
1. **HTML文件**: 直接在浏览器中打开
2. **报告文件**: 使用Markdown阅读器或文本编辑器
3. **数据文件**: 使用Excel、Python pandas等工具分析

### 分享结果
- HTML文件可直接分享给其他用户
- 报告文件适合包含在文档中
- 数据文件便于进一步分析处理

### 备份重要结果
对于重要的测试结果，建议：
- 重命名添加日期标识
- 移动到专门的备份目录
- 添加说明文档记录测试条件
